first steps

We started this project with one goal in mind, bring the best landing page for the best food recipe app you can find in iOS and we did it combining CSS3 animations and Responsive Web Design.

The result, a site that adjusts itself to any resolution and have a max width for some of those crazy screens that go beyond 1440p, we created different images for each asset that we got in the design so we could serve them if they were needed.

This boost the performance of the site by a lot, check our performance review to learn more, you can check the site and how it looks below.

the tools

We handle all the animations using custom events that ensure the behavior across all browsers. To meet the goal of a responsive site, we asked for different images to our professional photographer and used font icons to decrease the load size in smaller resolutions/devices

Second, we used our custom framework "Element CSS" which help us make the greatest sites without overthinking grids and doing unnecessary code.

And third, we created the code to be SEO compliant in every way, to run and load fast.

amazing performance

We achieve a 60fps framerate to have smooth transitions on every effect, this has a meaning, most LCD screens on smartphones, tablets and computers work at 60 hertz that's why we need to have a framerate equal to it so the customer don't encounter with "lag", we have two steps to ensure this.

We disable some important events that create a constant overload on the CPU to render something that it's already rendered on scroll.

We load and show only what the client is looking at, no need to load fancy animations and effects if you are not looking at them, this give us the advantage of being selective with more crucial enviroments like 3G data conection where you cannot wait to load a site more than 6 seconds.

Everyone in the company now has more time to invest in expanding our core business thanks to the time that Hashlabs system is giving us.
